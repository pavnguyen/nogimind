# Deep UX/UI Direction

NoGi Mind is for users who already have basic no-gi context. The app should feel like a technical cockpit, not a beginner course or personal tracker.

## Product Posture

- Lead with search, skill diagnosis, contact maps, and branches.
- Keep beginner explanation available, but not dominant.
- Prioritize exact execution: my body part, opponent body part, side, direction, pressure, cue, and mistake.
- Treat the skill detail page as the main technical manual.
- Keep advanced information collapsed until the user asks for it.

## Primary User Jobs

1. Debug a failing technique.
2. Check exact body-to-body contacts before drilling.
3. Compare branches when the opponent reacts.
4. Study modern systems without losing prerequisites and safety.
5. Search a cue quickly during training.

## App Structure

Navigation should stay problem-oriented:

- Train: dashboard, search, skills, details, troubleshooter.
- Map: positions, chains, escape maps, game tree.
- Systems: modern systems, archetypes, mastery, learning path.
- Reference: concepts, glossary, settings, philosophy.

## Skill Detail Order

1. Quick Start.
2. Money Details.
3. Contact Map.
4. Ready Check.
5. Key Terms.
6. High-Level Details.
7. Understand.
8. Adapt.
9. Next Step.

## Data Clarity Standard

Body-to-body data should read like:

- My right hand pulls the opponent's left wrist across centerline.
- Tay phải của tôi kéo cổ tay trái của đối thủ qua centerline.
- Ma main droite tire le poignet gauche de l'adversaire au-delà de la ligne centrale.

Avoid:

- Control the arm.
- Apply pressure.
- Turn the angle.
- Finish the choke.
